# 6. Методика применения

**Статус**: ✅ Черновик готов
**Источник**: Новый материал
**Объём**: ~500-600 слов

---

## Задачи раздела
1. Описать алгоритм валидации с использованием IIEE
2. Определить требования к входным данным
3. Представить этапы методики

## Алгоритм валидации

### Этап 1: Подготовка данных
1. Получить прогноз концентрации морского льда (SIC)
2. Получить наблюдаемые данные (референсные)
3. Привести к единой пространственной сетке
4. Применить порог 15% для бинаризации

### Этап 2: Расчёт компонентов
1. Для каждой ячейки сетки:
   - $c_{\text{прогноз}} = 1$ если SIC$_{\text{прогноз}}$ > 15%, иначе 0
   - $c_{\text{набл}} = 1$ если SIC$_{\text{набл}}$ > 15%, иначе 0
2. Рассчитать OE и UE:
   - OE = сумма площадей ячеек, где $c_{\text{прогноз}} = 1$ и $c_{\text{набл}} = 0$
   - UE = сумма площадей ячеек, где $c_{\text{прогноз}} = 0$ и $c_{\text{набл}} = 1$
3. Рассчитать IIEE, AEE, ME

### Этап 3: Интерпретация результатов
1. Сравнить IIEE с референсными значениями
2. Проанализировать соотношение AEE/ME
3. Если ME > AEE → проблема с положением границы
4. Если AEE > ME → систематическая ошибка площади

### Этап 4: Рекомендации
1. Для навигационных задач: приоритет моделям с низким ME
2. Для климатических задач: допустим приоритет низкого AEE

## Требования к данным

| Параметр | Требование |
|----------|------------|
| Пространственное разрешение | 1-25 км (в зависимости от задачи) |
| Временное разрешение | суточное или выше |
| Источник прогноза | Нейросетевая модель (IceNet и др.) |
| Источник наблюдений | Данные ДЗЗ (независимые от обучающей выборки) |
| Область | Акватория СМП / Баренцево море |

## Схема методики

```
[Прогноз SIC] → [Бинаризация 15%] → [Расчёт OE, UE]
                                           ↓
[Наблюдения SIC] → [Бинаризация 15%] → [IIEE = OE + UE]
                                           ↓
                                    [AEE = |OE - UE|]
                                    [ME = 2·min(OE, UE)]
                                           ↓
                                    [Интерпретация]
                                           ↓
                                    [Рекомендации для НГО]
```

## Текст раздела

На основании проведённого анализа метрик и обоснования приоритета ME для навигационных задач представляется методика валидации нейросетевых прогнозов ледовой обстановки.

### Этап 1. Подготовка данных

Входными данными для методики являются:

**Прогнозные данные** — результаты работы нейросетевой модели прогнозирования концентрации морского льда (SIC). Модель может быть основана на различных архитектурах (U-Net, ConvLSTM, LSTM и др.). Прогноз должен содержать значения SIC в диапазоне [0, 100%] для каждой ячейки пространственной сетки.

**Референсные данные** — наблюдаемые значения SIC из независимого источника. Критически важно, чтобы референсные данные не использовались при обучении модели [@anderssonSeasonalArcticSea2021]. Рекомендуемые источники:
- данные пассивного микроволнового зондирования (AMSR2, SSM/I);
- данные радаров с синтезированной апертурой (Sentinel-1, RADARSAT);
- ледовые карты ААНИИ [@afanasevaMetodikaSostavleniyaLedovyh2019].

**Приведение к единой сетке**. Прогнозные и референсные данные приводятся к единой пространственной сетке методом билинейной интерполяции. Разрешение сетки определяется задачей: 1–5 км для тактической навигации, 10–25 км для стратегического планирования.

### Этап 2. Бинаризация и расчёт компонентов

Для каждой ячейки сетки выполняется бинаризация по порогу 15%:

$$c_i = \begin{cases} 1, & \text{если SIC}_i > 15\% \\ 0, & \text{если SIC}_i \leq 15\% \end{cases}$$

Далее рассчитываются компоненты ошибки:

$$\text{OE} = \sum_{i \in A} S_i \cdot \mathbb{1}[c_{\text{прогноз},i} = 1 \land c_{\text{набл},i} = 0]$$

$$\text{UE} = \sum_{i \in A} S_i \cdot \mathbb{1}[c_{\text{прогноз},i} = 0 \land c_{\text{набл},i} = 1]$$

где $S_i$ — площадь ячейки $i$, $A$ — область интереса, $\mathbb{1}[\cdot]$ — индикаторная функция.

На основе OE и UE вычисляются итоговые метрики:

$$\text{IIEE} = \text{OE} + \text{UE}$$
$$\text{AEE} = |\text{OE} - \text{UE}|$$
$$\text{ME} = 2 \cdot \min(\text{OE}, \text{UE})$$

### Этап 3. Интерпретация результатов

Результаты валидации интерпретируются по следующей схеме:

| Соотношение | Диагноз | Рекомендация |
|-------------|---------|--------------|
| ME/IIEE < 0.5 | Доминирует ошибка площади | Модель пригодна для навигации; возможна коррекция систематического смещения |
| ME/IIEE ≥ 0.5 | Доминирует ошибка положения | Модель требует доработки для навигационных задач |
| OE >> UE | Консервативный прогноз | Приемлемо для навигации; экономические потери |
| UE >> OE | Оптимистичный прогноз | Опасно для навигации; требуется коррекция |

Дополнительно рекомендуется анализировать пространственное распределение ошибок OE и UE для выявления регионов с систематическими проблемами прогнозирования.

### Этап 4. Формирование рекомендаций

На основании результатов валидации формируются рекомендации для системы НГО:

1. **Выбор модели**. При наличии нескольких моделей-кандидатов выбирается модель с минимальным ME при условии приемлемого IIEE.

2. **Определение области применимости**. Для каждой модели определяются регионы и сезоны, где ME находится в допустимых пределах.

3. **Квантификация неопределённости**. Значения OE и UE используются для оценки вероятности ошибки прогноза в конкретном районе.

4. **Интеграция с экспертной оценкой**. Результаты автоматической валидации предоставляются ледовому аналитику для принятия финального решения о пригодности прогноза.

### Требования к реализации

Для практического применения методики необходимо обеспечить:

| Параметр | Требование |
|----------|------------|
| Пространственное разрешение | 1–25 км |
| Временное разрешение | суточное или выше |
| Область валидации | акватория СМП (28 районов по классификации ААНИИ) |
| Независимость данных | референсные данные не должны пересекаться с обучающей выборкой |
| Формат выходных данных | IIEE, AEE, ME, OE, UE в км²; карты пространственного распределения ошибок |

Методика может быть реализована в виде программного модуля для интеграции в существующие ГИС НГО.
